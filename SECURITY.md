## Правила добавления зависимостей

1. Только официальные пакеты с PyPI
2. Проверять количество загрузок и дату последнего обновления
3. Не устанавливать пакеты из GitHub напрямую без проверки кода
4. После установки запускать: `pip show <package>` и читать Location

## Правила работы с секретами

1. Все секреты только в `.env`
2. `.env` в `.gitignore` — никогда не коммитить
3. Fallback значения в коде — только пустые строки `""`
4. Ротация токенов при любом подозрении на компрометацию

## Правила проверки кода перед релизом

```bash
# Проверить нет ли секретов в коммите
git diff --staged | grep -E "AAAF|gsk_|86864|30611"

# Проверить нет ли рекламных ссылок
grep -rn "joinchat\|subscribe\|startgroup" scripts/ --include="*.py"

# Проверить внешние домены
grep -rn "https://" scripts/ --include="*.py" | grep -v "api.telegram.org\|api.groq.com\|localhost"
```

## Единственная точка входа

Запускать только `scripts/main.py`.
Никогда не запускать `scripts/bot.py` или `scripts/offer_parser.py` — они в архиве.
Два экземпляра main.py = конфликт 409.
